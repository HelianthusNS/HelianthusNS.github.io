Open-Graph Questions

i. What is the use of the open graph protocol?

The purpose of open graph protocol is to transform a webpage to create a way for the pages to store metadata that had contents of the page by using tags and thus turning it into a rich object link preview. It was originally developed by Facebook to "transform websites into a rich object in a social graph". In basic terms the open graph protocol lets the webpage have a visual preview when you post the link on sites to interact with the social media platforms

ii. What are the open graph tags inserted in an HTML document?

  1. <meta property="og:title" content="Scossa by Sangiovanni" />
  2. <meta property="og:type" content="music.song" />
  3. <meta property="og:url" content="https://open.spotify.com/track/7aoU4HfD4yc1LhpyldWFtm?si=ba5474933f47449c" />
  4. <meta property="og:image" content="https://i.scdn.co/image/ab67616d00001e021129556a8ba0a3a9c66ba67c" />

The open graph tag fields inserted in an HTML document are og:title, og:type, og:url, an og:image. All of them are in the format of <meta property="og:X" content="Y" />

iii. What prefix is used for Twitter's version of Open Graph Tags?

<meta name="twitter:card" content="summary_large_image"/>, Thus the prefix used is twitter instead of og and name is used instead of property for the twitter Open Graph Tags
